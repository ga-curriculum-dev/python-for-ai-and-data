# Introduction to Python and Jupyter Notebooks — Instructor Guide

**Module Time:** 120 minutes

---

## What This Module is About

This module introduces learners to Python and Jupyter Notebooks by guiding them through environment setup, basic notebook interaction, and reproducible project workflows. It establishes the technical and conceptual foundation for all subsequent data and AI work in the course.

---

## Core Objectives

- Prepare learners to work confidently in a Python and Jupyter environment  
- Introduce notebooks as reproducible analytical documents  
- Establish professional habits around documentation and project structure  
- Familiarise learners with Git and GitHub as core tools for versioning work  

---

## Must-Hit Topics

- Installing and verifying Python and Jupyter via Anaconda  
- Navigating the Jupyter Notebook interface  
- Code cells vs. markdown cells  
- Reproducible project folder structures  
- Basic Git and GitHub workflows  
---

**Timing Note:**  
The time allocations in this instructor guide mirror the session breakdown in the student README. Timings are approximate and may be adjusted based on cohort needs.

---

## 1. Environment Setup with Anaconda
**(Time: ~20 minutes)**

- **Purpose:**  
  Ensure all learners have a working and consistent Python environment before progressing.

- **Talking Points:**
  - Why environment setup matters for reproducibility  
  - What Anaconda includes (Python, Jupyter, package management)  
  - Differences learners may see across operating systems  
  - How to troubleshoot common installation issues  

- **Teaching Tips:**
  - Pause frequently to let learners catch up  
  - Ask learners to confirm visually that Jupyter launches successfully  
  - Reassure learners that installation issues are normal at this stage  

---

## 2. Introduction to Jupyter Notebooks
**(Time: ~25 minutes)**

- **Purpose:**  
  Introduce Jupyter Notebooks as the primary workspace for analysis and documentation.

- **Talking Points:**
  - What a Jupyter Notebook is and why it’s widely used in data roles  
  - Code cells vs. markdown cells  
  - How outputs are generated and displayed  
  - The role of the kernel  

- **Teaching Tips:**
  - Live-demo creating and running a simple notebook  
  - Encourage learners to experiment without fear of breaking anything  
  - Reinforce that notebooks combine analysis *and* communication  

---

## 3. Writing and Documenting Code
**(Time: ~25 minutes)**

- **Purpose:**  
  Help learners practise executing simple Python code and documenting it clearly.

- **Talking Points:**
  - Running basic Python expressions  
  - The importance of readable, well-documented notebooks  
  - Using markdown for titles, explanations, and structure  

- **Teaching Tips:**
  - Emphasise clarity over correctness at this stage  
  - Praise good documentation habits early  
  - Encourage learners to narrate what their code is doing in plain language  

---

## 4. Reproducible Project Structure
**(Time: ~30 minutes)**

- **Purpose:**  
  Introduce organisational habits that support scalable and professional data work.

- **Talking Points:**
  - Why folder structure matters in real-world projects  
  - Separating notebooks, data, and documentation  
  - How this structure supports collaboration and reuse  

- **Teaching Tips:**
  - Show examples of messy vs. clean project structures  
  - Reinforce consistency over perfection  
  - Connect structure to future labs and the capstone project  

---

## 5. Version Control with Git and GitHub
**(Time: ~15 minutes)**

- **Purpose:**  
  Introduce version control as a core professional skill, not an optional add-on.

- **Talking Points:**
  - What Git tracks and why it’s useful  
  - What GitHub adds on top of Git  
  - High-level overview of commit and push workflows  

- **Teaching Tips:**
  - Keep this conceptual — details will be reinforced in later modules  
  - Emphasise GitHub as a portfolio and collaboration tool  
  - Normalize confusion and reassure learners they will practise this repeatedly  

---

## Wrap-Up Reflection
**(Time: ~5 minutes)**

- **Key Takeaways:**
  - A reliable environment is the foundation of all data and AI work  
  - Jupyter Notebooks combine computation, documentation, and communication  
  - Reproducibility starts with small habits: structure, markdown, and version control  

- **Next Steps:**  
  Learners move into the lab to set up their environment, create their first notebook, and publish their work to GitHub.

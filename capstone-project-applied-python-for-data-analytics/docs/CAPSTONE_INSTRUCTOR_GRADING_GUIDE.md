# CAPSTONE PROJECT – INSTRUCTOR GRADING GUIDE  
*Instructor Use Only – Not for Distribution to Learners*

---

## Purpose

This guide supports consistent grading across instructors while preserving the open-ended nature of the Capstone Project.

The official grading rubric and scoring system are defined in the Capstone Project Brief. This document provides additional guidance on how to interpret and apply those criteria consistently.

The capstone evaluates applied competency, analytical reasoning, and professional execution — not perfection.

---

## 1. High-Level Grading Alignment

Use the official 1–4 scoring scale per category as outlined in the Capstone Brief.

This guide clarifies what “Meets” versus “Exceeds” expectations typically look like in practice.

---

## 2. Interpretation Guidance by Category

---

### A. Problem Framing & Scope

**Look for:**

- Clear analytical objective or business question  
- Logical connection between problem and dataset  
- Defined scope (not overly broad or vague)

**Meets Expectations (3)**

- Clear objective  
- Reasonable scope  
- Analysis direction aligns with question  

**Exceeds Expectations (4)**

- Strong business framing  
- Explicit justification for scope decisions  
- Demonstrates awareness of trade-offs or assumptions  

**Common Pitfalls**

- Vague objective (“analyze the data”)  
- No clear analytical focus  
- Scope too broad for dataset size  

---

### B. Data Handling & Preparation

**Look for:**

- Proper data loading  
- Cleaning steps explained  
- Handling of missing values  
- Reproducible transformations  

**Meets Expectations (3)**

- Data prepared correctly  
- Cleaning steps documented  
- No major data errors  

**Exceeds Expectations (4)**

- Thoughtful validation checks  
- Clear reasoning behind transformations  
- Appropriate feature engineering  

**Common Pitfalls**

- Silent data cleaning  
- Ignoring missing values  
- Transformations not reproducible  

---

### C. Exploratory Data Analysis (EDA)

**Look for:**

- Minimum two professional visualizations  
- Clear labeling (titles, axes, legends)  
- Insights documented in markdown  

**Meets Expectations (3)**

- Visualizations support analysis  
- Insights described clearly  

**Exceeds Expectations (4)**

- Visualizations drive meaningful insight  
- Strong narrative flow  
- Interpretation connects clearly to business objective  

**Common Pitfalls**

- Charts without explanation  
- Decorative plots with no analytical value  
- Incorrect interpretation of patterns  

---

### D. Modeling & Evaluation

**Look for:**

- Proper train-test split  
- Baseline model implemented correctly  
- Appropriate evaluation metrics  
- Model saved (.joblib)

**Meets Expectations (3)**

- Model trains successfully  
- Metrics calculated correctly  
- Interpretation present  

**Exceeds Expectations (4)**

- Metrics contextualized in business terms  
- Reflection on model limitations  
- Clear understanding of overfitting/underfitting  

**Common Pitfalls**

- No train-test split  
- Reporting metrics without interpretation  
- No saved model artifact  

---

### E. Professionalism & Reproducibility

**Look for:**

- Notebook runs top-to-bottom without errors  
- Clean structure and markdown documentation  
- Repository organized per required structure  
- README, Model Card, and requirements.txt included  
- Clear commit history  

**Meets Expectations (3)**

- Complete and runnable submission  
- Professional organization  

**Exceeds Expectations (4)**

- Industry-ready presentation quality  
- Strong documentation clarity  
- Clear reproducibility steps  

**Common Pitfalls**

- Notebook only runs due to cached variables  
- Missing documentation  
- Single large commit instead of development history  

---

## 3. AI Usage Guidance

Learners may use AI tools for:

- Debugging  
- Syntax clarification  
- Concept explanation  

Learners should demonstrate understanding of:

- Their analysis decisions  
- Their modeling approach  
- Their interpretation of results  

If interpretation appears generic or disconnected from analysis, instructors may probe during presentation to verify understanding.

---

## 4. Alignment with Official Scoring

This guide does not replace the official grading rubric in the Capstone Brief.

Use this document to:

- Reduce grading variability  
- Align expectations across sections  
- Support fair and consistent evaluation  

Final numeric scoring should follow the official rubric.

---

## 5. Instructor Reminder

The capstone is intentionally open-ended.

Avoid:

- Penalizing creative approaches  
- Expecting a specific modeling technique  
- Rewarding unnecessary complexity over clarity  

The goal is to evaluate applied reasoning, analytical competency, and professional communication.
